Stack: Vite + TypeScript + Tailwind CSS + shadcn/ui (confirmed by files such as tailwind.config.ts, components.json).

Goals:

Update the “Portfolio / Our Work” page to match the platform’s brand colors (White + Sky Blue), keeping full consistency with the overall design.

General improvements to the platform if missing (Arabic/English i18n + RTL, optimized images, SEO).

3) Portfolio Pages (Index + Details)
A) /portfolio (Index)

Header: Title, subtitle, overall metrics (e.g., total projects, industries served).

Filters/Search/Sort bar (shadcn/ui components):

Search by title, client, tags.

Filters: industry, tech, year, status.

Sort: newest, oldest, alphabetical, impact (KPI).

Grid of Cards: project cover image, name, client, sector, 3–4 short KPIs.

Each card → “Details” button linking to /portfolio/[slug].

Lightbox for gallery images.

B) /portfolio/[slug] (Details)

Hero Section: title, client, industry, duration, provided services.

KPIs Block: grid with KPI cards (value + short description).

Tech Stack Block: badges/icons (Flutter, ASP.NET, SQL Server, etc.).

Gallery Block: screenshots with lightbox.

Story Block: Challenge / Solution sections.

CTA: “Request Similar Project” → Contact page.

4) Data & i18n

Create client/src/data/portfolio.ts with structured objects { slug, title, client, sector, year, status, kpis[], tech[], coverImage, gallery[], summary_en, summary_ar, ... }.

i18n: Ensure translations for labels/buttons exist in JSON.

Auto-switch dir="rtl" for Arabic and ltr for English.

5) Components

PortfolioFilters.tsx: filter/search/sort bar.

PortfolioCard.tsx: project card with KPIs and hover animation.

KpiPill.tsx: reusable component for KPIs.

TechBadge.tsx: badge for technologies.
